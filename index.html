Job Applications
Claude is strong at writing, tailoring, and matching tone — but you own the facts.
📋
Step-by-Step Application Workflow
Claude: High
Paste the job description in full. Don't summarize it. The more Claude sees, the better the keyword and tone match.
Paste your current CV/resume. Tell Claude what's accurate and what to work with.
Ask Claude to identify gaps and mismatches between your profile and the JD before writing anything.
Ask for a tailored cover letter with explicit tone, length, and structure instructions.
Ask Claude to suggest specific bullet rewrites for your CV that match this JD's language.
Run a final audit: ask Claude "What assumptions did you make that I should verify?"
Prompt Template — Cover Letter
I'm applying for this role: [paste full JD] My background: [paste CV or key bullet points] Write a cover letter that: - Is [200 / 300 / 400] words - Opens with a specific hook (not "I am excited to apply") - Addresses the top 3 requirements from the JD directly - Tone: [confident and direct / warm and collaborative / formal] - Does NOT use the phrases "passionate", "synergy", or "results-driven" - Closes with a clear call to action Do not fabricate experience I haven't mentioned. Flag anything you assumed.
Prompt Template — Gap Analysis First
Here is a job description: [paste JD] Here is my CV: [paste CV] Before writing anything, do this: 1. List the top 5 requirements in the JD 2. Rate how well my CV addresses each (Strong / Partial / Missing) 3. Tell me what to emphasize and what to NOT mention 4. Identify any language/keywords I should use that aren't in my CV Then ask me if I want to proceed to the cover letter.
Always verify: Every factual claim, date, title, and metric. Claude will write confidently but may reframe things beyond what's accurate. You are responsible for accuracy — misrepresentation on an application is your risk, not Claude's.
Never ask Claude to: Invent experience you don't have, fabricate references, or write skills you can't demonstrate in an interview.

Interview Prep Workflow
Claude: High
Prompt Template — Mock Interview
I'm interviewing for: [role + company type] My background: [2-3 sentence summary] Run a mock interview. Ask me 8 questions — a mix of: - 3 behavioral (STAR format expected) - 3 technical/role-specific - 2 difficult/unexpected questions they might throw After each answer I give, do this: 1. Rate it (Strong / Acceptable / Weak) 2. Identify what was missing 3. Give me a rewritten version that's stronger Be harsh. Don't tell me it was good if it wasn't.
Prompt Template — Salary Negotiation Prep
I've received an offer: [describe offer details] My target: [your desired comp] Their likely constraints: [anything you know about the company] Give me: 1. A negotiation strategy (what to ask for, what order) 2. Exact scripts for the negotiation call 3. The 3 most likely pushbacks and how to handle each 4. What NOT to say Note: I'll verify market rates separately — focus on strategy and language.
Resume Editing
Claude is excellent at rewriting bullets for impact — weak at knowing what's standard in your specific industry.
📄
Resume Audit & Rewrite Workflow
Claude: High
Paste your full CV. Tell Claude your target role and target industry.
Ask for a cold audit — what's weak, what's missing, what's generic.
Rewrite section by section. Don't ask for a full rewrite in one shot — quality degrades.
Ask Claude to apply the XYZ formula (Accomplished X, as measured by Y, by doing Z) to each bullet.
Ask for ATS keyword optimization for a specific JD after the human-facing version is done.
Prompt Template — Cold Audit
Here is my resume: [paste full CV] Target role: [job title] Target industry: [industry] Level: [junior / mid / senior / exec] Audit this resume ruthlessly. For each section: 1. What's working 2. What's weak or generic 3. What's missing entirely 4. Specific rewrites for the 3 worst bullets Do not compliment the good parts excessively. I want actionable critique.
Prompt Template — Bullet Rewrite
Rewrite these resume bullets using the XYZ formula: "Accomplished [X] as measured by [Y] by doing [Z]" Current bullets: [paste 4-6 bullets] Rules: - Start with strong action verbs (not "Responsible for" or "Helped") - Include metrics wherever possible — flag where I need to add them - Keep each bullet under 2 lines - Avoid: "leveraged", "spearheaded", "synergy", "passionate" - Industry: [your field]
Prompt Template — ATS Optimization
Here is a job description: [paste JD] Here is my resume: [paste resume] List: 1. Keywords in the JD that are missing from my resume 2. Keywords I have that match — good, keep them 3. Suggested edits to naturally incorporate missing keywords 4. Sections where keyword density is too low Do not stuff keywords unnaturally. Integrate them into real bullet content.
Always verify: Every metric you use in bullets. Every date and job title. Claude may round up or reframe things in ways that feel like embellishment. You must own the accuracy.
Coding & Debugging
Claude understands code well and explains clearly. It cannot execute code and misses environment-specific issues.
💻
Debugging Workflow
Claude: High
Paste the exact error message — full stack trace, not a summary.
Paste the relevant code block. If it spans files, say so and paste both.
State your environment: language version, framework, OS, key library versions.
Ask Claude to explain what the error means before suggesting a fix — this forces accuracy.
Ask "what could go wrong with this fix?" before implementing.
Test it. Never assume Claude's code runs without testing.
Prompt Template — Debugging
Language/framework: [Python 3.11 / Node 20 / React 18 / etc.] Environment: [local dev / Docker / AWS Lambda / etc.] Key libraries: [list versions] Error message: [paste full stack trace] Relevant code: [paste code block] Do this in order: 1. Explain what the error means in plain English 2. Identify the most likely cause (and 1-2 alternatives) 3. Give me the fix with comments explaining each change 4. Tell me what else could break because of this fix 5. Suggest a test to verify it's actually fixed
Prompt Template — Code Review
Review this code as a senior engineer would: [paste code] Focus on: 1. Logic errors or edge cases I've missed 2. Security issues (injection, auth, input validation) 3. Performance concerns 4. Readability — what's unclear 5. What I should test before shipping Give specific line-level feedback, not generic advice. Use [language], [framework], targeting [production / internal / prototype].
Prompt Template — Feature Implementation
I need to implement: [describe feature clearly] Existing codebase context: - Stack: [list stack] - Pattern we follow: [MVC / service layer / hooks / etc.] - Relevant existing code: [paste key files] Constraints: - Must not break: [list existing behavior] - Performance requirement: [if any] Give me: 1. The implementation plan before writing code 2. The code, with comments 3. Unit tests 4. What I should manually test that tests won't catch
Never deploy Claude's code directly. It cannot run it. It can't know your specific environment. Always test in staging first. Security-sensitive code (auth, payments, input validation) needs senior human review regardless.
Research
Claude is a synthesis engine, not a search engine. Use it to understand and structure — not to source facts.
🔍
Research Workflow
Trust: Medium
Use Claude to map the landscape — what are the key concepts, debates, and questions in this area?
Ask Claude what you should look for — what sources, what search terms, what databases.
Do your own sourcing via Google Scholar, official reports, news, primary sources.
Paste your found sources into Claude and ask it to synthesize, compare, and identify gaps.
Ask Claude to steelman opposing views — stress-test your conclusion.
Prompt Template — Topic Landscape
I'm researching: [topic] My purpose: [academic paper / business decision / personal understanding / article] My current knowledge level: [beginner / intermediate / expert] Give me: 1. The 5 most important concepts I need to understand 2. The key debates or disagreements in this field 3. What experts disagree on (and why) 4. The questions I should be asking that I probably haven't thought of 5. What search terms and source types I should use Do NOT give me citations — I'll find those myself. Focus on conceptual orientation.
Prompt Template — Synthesis from Pasted Sources
I've collected these sources/excerpts on [topic]: Source 1: [paste excerpt] Source 2: [paste excerpt] Source 3: [paste excerpt] Do this: 1. Identify the main claims each source makes 2. Where they agree 3. Where they contradict each other 4. What gaps remain — what no source addresses 5. What conclusion the evidence most supports Flag anything where you're extrapolating beyond what the sources say.
Prompt Template — Devil's Advocate
My current conclusion is: [state your position] Based on the research I've shared, steelman the opposing view. Give me the strongest possible case AGAINST my conclusion. Then tell me: 1. Which counterarguments are genuinely strong 2. Which are weak or easily dismissed 3. What I should acknowledge even if I still hold my position
Never ask Claude for citations you plan to use. It will generate plausible-sounding but often fabricated references. Always source independently and paste sources in for Claude to analyze.
Automation
Claude is strong at writing automation scripts, designing logic, and explaining APIs — weak at knowing your specific environment.
⚙️
Automation Scripting Workflow
Claude: High
Describe the task in plain English first. Don't jump to "write me the code" — describe what you want to happen.
Ask Claude to design the logic/flow before writing code. Catch design flaws early.
Specify your environment precisely: OS, language, installed tools, existing scripts it should connect with.
Ask for error handling and logging explicitly — Claude won't add it unless asked.
Test on dummy data first. Never run new automation on real/production data first.
Prompt Template — Automation Script
I want to automate: [describe the task in plain English] Environment: - OS: [Windows / Mac / Linux / Docker] - Language preference: [Python / Bash / JS / etc.] - Tools available: [list installed tools] - Runs: [manually / cron / triggered by X] Before writing code: 1. Describe the approach you'd take 2. List assumptions you're making 3. Flag any risks I should know about Then write the script with: - Inline comments explaining non-obvious logic - Error handling for the most likely failure points - A test mode that doesn't affect real data - Logging so I can see what it did
Prompt Template — Zapier / Make / n8n Logic Design
I want to build a no-code automation that: [describe trigger → action chain in plain English] Tools I have access to: [Zapier / Make / n8n / etc.] Apps involved: [list apps] Design the workflow for me: 1. Exact trigger → filter → action chain 2. What data needs to be mapped between steps 3. Where it will likely break and how to handle it 4. What I should test before going live Don't assume I know the tool deeply — explain each step.
Prompt Template — API Integration
I need to integrate with: [API name] My goal: [what you want to do with it] My language: [Python / JS / etc.] Here is the relevant API documentation: [paste docs or endpoint spec] Write me: 1. The authentication setup 2. The specific API call(s) I need 3. Response parsing for the data I care about 4. Error handling for common API errors (rate limits, auth failures, timeouts) Note: use only what's in the docs I've provided. Don't assume endpoints that aren't listed.
Always verify: API calls against real documentation (Claude may use outdated or slightly wrong endpoint patterns). Test scripts on dummy data. Review any file system or data manipulation operations line by line before running.
Media / Press Research
Claude is strong at drafting, framing, and analysis. Weak at finding current coverage — use web search for that.
📰
Press & Media Workflow
Verify: High
For press release writing: Claude is excellent. Paste your key facts, audience, and objective.
For media list research: Claude knows publication types, not current journalist beats. Use it for strategy, not contact lists.
For analyzing existing coverage: Paste the articles into Claude and ask for analysis, angle comparison, or response drafting.
For media monitoring: Use real tools (Google Alerts, Mention, Meltwater). Claude can help you define what to monitor.
Prompt Template — Press Release
Write a press release for: What happened: [the announcement/news] Company/org: [name + one-line description] Target outlets: [tech / business / local / trade press] Key message: [what you want people to remember] Quote from: [name + title] — suggested quote theme: [what it should convey] Format: - Standard inverted pyramid structure - Headline + subheadline - Dateline + city - 400-500 words - Boilerplate at the end: [paste or say "draft one"] Do not fabricate specific statistics. Flag where I need to add data.
Prompt Template — Media Pitch
I need to pitch this story: [describe your story in 2-3 sentences] Target journalist beat: [technology / finance / sustainability / etc.] Target publication type: [national newspaper / trade mag / newsletter / podcast] Why it's newsworthy NOW: [the peg / news hook] Write a cold pitch email: - Subject line (3 options) - 150-word pitch body maximum - Clear "what's in it for their readers" angle - One specific ask Tone: journalist-to-journalist, not PR-to-journalist. No superlatives. No "revolutionary" or "game-changing".
Prompt Template — Coverage Analysis
Here are [X] articles about [topic / company / event]: [paste articles or excerpts] Analyze: 1. What angle/narrative is dominant across coverage 2. What's being missed or underreported 3. What the sentiment is (and whether it's shifting) 4. If this is about my company/topic: what should our response strategy be? 5. Who seems to be driving the narrative — and who we should be talking to
Prompt Template — Spokesperson Prep / Crisis Comms
Situation: [describe the issue or crisis] Our current position: [what the organization says happened] Key stakeholders: [media / customers / investors / employees] Give me: 1. The 3 most likely hostile questions from journalists 2. Approved holding statements for each 3. What NOT to say and why 4. The core message we must land regardless of what they ask 5. Red lines — what would escalate this if mishandled
For media monitoring and finding journalist contacts: Claude cannot search the web in real-time (unless you've enabled web search in Claude.ai). Do not ask Claude for current journalist names, emails, or recent coverage — it may give outdated or fabricated information.

The 3 Trust Rules
Trust Claude on: Process, structure, language, logic, explanation of things it was trained on well. The "shape" of thinking is reliable even when specific facts aren't.
Verify when: There's a specific fact, number, name, citation, or current event in the answer. The more specific the claim, the more likely it needs checking.
Never act on alone: Legal documents, medical decisions, financial trades, production deployments, government submissions. Claude is a starting point, not an ending point for these.

Power User Multi-AI Workflow
Claude: All reasoning, writing, analysis, document review, code understanding. The thinking layer.
ChatGPT: Running and testing code (Code Interpreter). Image generation. Voice tasks.
Gemini: Anything in Google Workspace. Video/audio analysis. Real-time Google-integrated tasks.
Perplexity / Claude with search: Real-time research with citations. Don't ask any AI to cite from memory.
The meta-rule: Use Claude to think and write. Use ChatGPT to execute and generate visuals. Use Gemini for Google-native tasks. Combine them — they're tools, not allegiances.
Daily Professional Workflow Using Claude
Morning: Paste your task list. Ask Claude to identify which items need the deepest thinking and suggest what to tackle first.
Research phase: Ask Claude for conceptual orientation. Do your own sourcing. Paste sources back in for synthesis.
Drafting: Use Claude for first drafts of everything written — emails, proposals, code comments, documentation.
Review pass: Paste your own draft back in. Ask Claude to be a harsh critic. "What's weak? What's missing? What would make this fail?"
Decisions: "Here's my reasoning. Find the flaws." Use Claude as a sparring partner, not a validator.
Before shipping: Human review of everything — code, content, legal docs. Claude's output is high-quality raw material, not a finished product.
